Buffering

Allow for 128 buffers per client. If 129 is requested, return NO_MORE_CAPACITY.
If 128 buffers are occupied, the server tells the client when 1 is freed. Send message RELEASED.

Client will wait until it receives RELEASED to send more messages.

Client buffers that don't reach valid message state will be erased.  They will be erased after 30 seconds. The client will be alerted with MESSAGE_ERROR and the buffer will be dumped.

Each time a message is ready to be sent, the server will send the client MESSAGE PROCESSED. The client will mark the earliest message sent with a processed state.

The entire buffer will be of x bytes. Each user will be given 65536 bytes to occupy. This will be broken down to 128 segments. Each segments will be of 512 bytes. Each user will always have one buffer.

The maximum size of a message is of 65536 - header size (To be defined).
