Message Header (8 bytes)

Each client message should be composed as:

HEAD_BITS  TYPE     RECEIVER_KEY    PAYLOAD_LENGTH  PAYLOAD
1 byte     1 byte   4 bytes         2 bytes         0 - 65024 bytes

HEAD_BITS = 11111111
Simple sequence of bits.

TYPE = USER (1) | GROUP (2) | AUTH_KEY (3) | SEND_REQUEST (4) | ACCEPT_REQUEST (5) | UPDATE (6)
only USER for now.

RECEIVER_KEY = The destinatory USER | The destinatory GROUP | The source USER | THe source GROUP
The username + key must exit in the database.
The group should be registered with the sender key for them to send it.
Skipped reading on AUTH_KEY.

PAYLOAD_LENGTH = n
length of payload in bytes.

PAYLOAD = message
message to be sent.
message is empty for INITIATE, CONCLUDE and CANCEL.
65024 bytes of payload subtracting 512 bytes to avoid issues with buffering.
